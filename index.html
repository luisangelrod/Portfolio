BerriosPay

Enterprise payment kiosk system for Empresas Berrios Inc. — full hardware/software integration with resilience-first architecture.

Architecture Overview

Presentation Layer
React 19 + TypeScript + TailwindCSS
Kiosk Shell
WPF + WebView2 (Hardware Bridge)
API Layer
.NET 10 Minimal APIs + Clean Arch
Data + Cloud
Azure SQL + App Service + Bicep IaC

The Challenge

Empresas Berrios needed to modernize payment collection — replacing manual processes with a unified system across kiosk, web, and mobile. The system had to integrate three hardware devices, work offline, prevent duplicate payments, and serve bilingual customers. I was the sole engineer.

Architecture Decisions

  • Polly circuit breaker + retry for hardware resilience
  • Idempotency layer preventing duplicate payments
  • Offline queue with automatic sync recovery
  • WPF shell bridging web app to native hardware

Sole Engineer

  • Owned the entire engineering lifecycle — frontend, backend, kiosk shell, database schema, cloud infrastructure (Azure), and hardware integration layer. Delivered a scope that enterprise teams typically staff with 4-6 engineers.
